Enterprise-level credential exfiltration response playbook

Enterprise Security Operations Runbook

Version: 2026.1

Mission: Rapid Detection, Containment, and Reporting of Credential-Based Exfiltration.

Documentation Purpose:

This runbook demonstrates my ability to document complex, multi-phase security operations workflows that integrate multiple enterprise security tools into a cohesive response strategy.

Phase 1: Detection & Triage (Splunk ES)

The Enterprise Security Dashboard uses Splunk ES to aggregate logs from across organizational divisions.

Trigger:

  • A "Notable Event" is generated in Splunk

Logic:

  • Multiple failed logins followed by a successful login from a non-reputable IP (Geo-fenced alert)
  • Targeting a critical system or API gateway

Analyst Action:

  • Open the Incident Review dashboard in Splunk ES
  • Validate the risk score (Risk-Based Alerting)
  • Use Splunk Search Processing Language (SPL) to pivot and identify if the compromised account has accessed any sensitive data stores

Phase 2: Investigation & Context (Cortex XSOAR)

Once the alert is validated, the workflow transitions to Cortex XSOAR (SOAR) for automated enrichment.

Automation Action:

  • The Splunk alert triggers a XSOAR Playbook

Enrichment:

  • XSOAR automatically queries threat intelligence feeds for the suspicious IP
  • Queries identity management systems to pull the user's role and access level

Analyst Action:

  • Review the "Work Plan" in XSOAR
  • If the IP is confirmed malicious, the analyst clicks "Confirm/Quarantine"

Phase 3: Containment & Remediation (XSOAR & XDR)

The automated response occurs here, leveraging the Zero Trust architecture of the enterprise environment.

Containment Steps:

  • Block IP: XSOAR pushes a command to the Next-Generation Firewall to block the malicious IP globally
  • Disable Account: Automatically disables the compromised account
  • Isolate Endpoint: Uses Endpoint Detection and Response (EDR/XDR) to isolate the host machine from the network to prevent lateral movement

Phase 4: Compliance Validation (Configuration Management)

Following an incident, the environment must verify that systems involved still meet security baselines.

Remediation Action:

  • Configuration management tool is triggered to scan the affected environment

Baseline Check:

  • Verifies the system against Security Technical Implementation Guides (STIGs)
  • Ensures no security controls were deactivated during the breach

Automated Hardening:

  • If configuration drift is detected, the tool automatically reapplies the secure configuration

Result:

  • Clean/compliant status is pushed back to the SIEM for the analytic report

Phase 5: Reporting & Analytic Feedback (Security Dashboard)

This phase documents the event for executive and regulatory oversight.

Reporting Action:

  • Generate Report: SIEM aggregates the timeline from Detection → Enrichment → Remediation
  • Compliance Requirement: The report is automatically formatted for executive leadership and regulatory bodies
  • Analytic Support: Data is fed into trend analysis to adjust future detection logic across the enterprise

Documentation Value Delivered:

  • Clarity: Complex multi-tool workflow made understandable
  • Actionable: Step-by-step instructions for security analysts
  • Compliant: Meets regulatory reporting requirements
  • Scalable: Can be adapted across enterprise divisions
  • Auditable: Clear documentation trail for investigations

Tools & Technologies Documented:

  • SIEM: Splunk Enterprise Security
  • SOAR: Cortex XSOAR
  • Firewall: Palo Alto Networks NGFW
  • EDR/XDR: Microsoft Defender
  • Compliance: NIST/FISMA frameworks
  • Identity: Active Directory/Okta
